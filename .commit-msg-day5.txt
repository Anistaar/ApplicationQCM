feat(pedagogy): Day 5 - OpenQ questions ouvertes + LLM prompts + audio feedback

WCAG & Pedagogie Enhancement:
- New question type OpenQ: Open-ended questions with free-text answers
- Binary validation: ALL keywords present (fuzzy Levenshtein distance <=2) = correct, else incorrect
- No partial scores (spartiate approach): either right or wrong
- Audio feedback: Success beep 800Hz 0.3s (Web Audio API) on correct answer only
- No sound on error (silent, focused on positive reinforcement)

OpenQ Format:
- Syntax: OpenQ || Question || keyword1,keyword2,keyword3 || Course reference || Explanation
- Keywords: 3-5 essential concepts (lowercase, technical terms)
- Course reference: Exact 1-2 sentence quote from course material
- Validation: Tokenize user text, check ALL keywords present (exact or fuzzy match)
- Fuzzy matching: Accepts typos with Levenshtein distance <=2

UI Implementation:
- Textarea: min-height 150px, character counter, resize vertical
- Focus ring on focus, disabled state styling
- Feedback correct: Green background, checkmark, course reference (collapsible details)
- Feedback incorrect: Red background, X mark, missing keywords list, course reference, advice
- ARIA: aria-describedby, role textbox, aria-live polite for feedback

Validation Algorithm:
- Tokenize: lowercase, remove punctuation, split by spaces, filter tokens <3 chars
- Keyword matching: Check each keyword appears in tokens (exact match or Levenshtein <=2)
- Binary result: ALL keywords found = true, else false
- Levenshtein implementation: Dynamic programming matrix O(n*m)
